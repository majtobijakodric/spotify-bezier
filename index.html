Animations

1) Plain image (PNG)

Render the Spotify logo using a plain PNG image with the <img> tag. No animation required.

  • Pros: Simple, widely supported, fast to implement
  • Cons: Fixed resolution, larger file size at high quality
<img 
  src="assets/spotify-png.png" 
  alt="Spotify Logo"
  width="200"
  height="200"
/>

2) SVG hover animation

Inline SVG with CSS transitions. Default fill is white, on hover it scales to 1.2× and transitions to Spotify green.

  • Pros: Smooth transitions, scalable, no JS needed
  • Cons: Requires inline SVG for fill animations
<svg class="svg-hover-logo" viewBox="0 0 48 48">
  <circle cx="24" cy="24" r="24"/>
  <path d="M34.4 21.4c-6.2-3.7-16.4-4-22.3-2.2..."
        class="svg-hover-waves"/>
</svg>
.svg-hover-logo {
  fill: #fff;
  transition: transform 0.3s ease-in-out,
              fill 0.5s ease;
}
.svg-hover-logo:hover {
  transform: scale(1.2);
  fill: #1ED760;
}

4) Gradient sweep shimmer

Inline SVG filled with an animated linear gradient that creates a "shimmer" sweep effect across the logo.

  • Pros: Eye-catching, modern effect
  • Cons: Requires JS for gradient animation
<svg viewBox="0 0 48 48">
  <defs>
    <linearGradient id="shimmer-grad">
      <stop offset="0%" stop-color="#fff"/>
      <stop offset="50%" stop-color="#1ED760"/>
      <stop offset="100%" stop-color="#fff"/>
    </linearGradient>
  </defs>
  <path fill="url(#shimmer-grad)" .../>
</svg>
function animateShimmer() {
  const grad = document.getElementById('shimmer-grad');
  let offset = -100;
  function step() {
    offset = (offset + 1) % 200 - 100;
    grad.setAttribute('gradientTransform', 
      `translate(${offset / 100}, 0)`);
    requestAnimationFrame(step);
  }
  step();
}

5) Stroke-draw reveal

SVG paths animated with stroke-dasharray and stroke-dashoffset to create a "drawing" effect over ~1.2 seconds.

  • Pros: Pure CSS animation, no JS required
  • Cons: Works best with stroke-based paths
<svg class="stroke-draw-logo" viewBox="0 0 48 48">
  <circle cx="24" cy="24" r="22" 
          class="stroke-draw-circle"/>
  <path class="stroke-draw-wave" d="..."/>
</svg>
.stroke-draw-circle {
  stroke: #fff;
  stroke-width: 2;
  fill: none;
  stroke-dasharray: 140;
  stroke-dashoffset: 140;
  animation: draw 1.2s ease forwards;
}
@keyframes draw {
  to { stroke-dashoffset: 0; }
}

6) Clip-path reveal loop

SVG logo revealed via an animated circular clip-path that expands from center, then loops smoothly.

  • Pros: Smooth, loopable, subtle effect
  • Cons: clip-path support varies in older browsers
<div class="clip-reveal-container">
  <svg class="clip-reveal-logo" viewBox="0 0 48 48">
    <path fill="#fff" d="..."/>
  </svg>
</div>
.clip-reveal-container {
  animation: clipReveal 2.5s ease-in-out infinite;
}
@keyframes clipReveal {
  0% { clip-path: circle(0% at 50% 50%); }
  50% { clip-path: circle(75% at 50% 50%); }
  100% { clip-path: circle(0% at 50% 50%); }
}

7) Glitch hover effect

On hover, duplicated SVG layers create a glitch effect with horizontal jitter and color offsets.

  • Pros: Eye-catching, trendy effect
  • Cons: Complex markup, can be distracting
<div class="glitch-container">
  <svg class="glitch-logo glitch-logo--main">...</svg>
  <svg class="glitch-logo glitch-logo--r">...</svg>
  <svg class="glitch-logo glitch-logo--g">...</svg>
</div>
.glitch-container:hover .glitch-logo--r {
  animation: glitchR 0.3s infinite;
  opacity: 0.8;
}
@keyframes glitchR {
  0%, 100% { transform: translate(-2px, 0); }
  50% { transform: translate(2px, 1px); }
}
